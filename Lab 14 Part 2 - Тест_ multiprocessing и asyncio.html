Вопрос 1

Для какого типа задач лучше всего подходит модуль multiprocessing?

A IO-bound (сетевые запросы, файлы)
B CPU-bound (вычисления, математика)
C Оба типа одинаково
D Только для работы с базами данных
multiprocessing создаёт отдельные процессы ОС, что позволяет задействовать несколько ядер CPU. Это даёт ускорение именно для CPU-bound задач.
Вопрос 2

Что такое GIL и как multiprocessing решает связанную с ним проблему?

A GIL — это сетевой протокол; multiprocessing его не использует
B GIL — Global Interpreter Lock; multiprocessing обходит его, создавая отдельные процессы с собственным интерпретатором
C GIL — глобальная блокировка памяти; multiprocessing её отключает
D GIL не существует в Python 3
GIL (Global Interpreter Lock) не позволяет нескольким потокам одновременно выполнять байт-код Python. Каждый процесс в multiprocessing имеет собственный интерпретатор и свой GIL, поэтому они выполняются параллельно.
Вопрос 3

Зачем используется multiprocessing.Queue?

A Для сортировки процессов по приоритету
B Для ограничения числа запущенных процессов
C Для передачи данных между процессами
D Для синхронного запуска процессов
Каждый процесс имеет собственную копию памяти. Queue — безопасный механизм межпроцессного обмена данными (IPC), позволяющий передавать результаты из дочернего процесса в родительский.
Вопрос 4

Почему нельзя использовать обычные глобальные переменные для обмена данными между процессами?

A Python запрещает глобальные переменные
B Глобальные переменные слишком медленные
C Каждый процесс работает с собственной копией памяти
D Глобальные переменные доступны только для чтения
При создании нового процесса ОС копирует всю память родительского процесса. Изменения переменных в дочернем процессе не видны в родительском — это разные адресные пространства.
Вопрос 5

Что делает метод Process.join()?

A Объединяет два процесса в один
B Запускает процесс
C Блокирует вызывающий процесс до завершения дочернего
D Принудительно завершает процесс
join() заставляет родительский процесс ожидать завершения дочернего. Без join() главный процесс может завершиться раньше, чем дочерние.
Вопрос 6

В чём преимущество Pool перед созданием отдельных Process на каждую задачу?

A Pool всегда быстрее
B Pool не требует if __name__ == '__main__'
C Pool создаёт фиксированное число процессов и распределяет задачи между ними
D Pool использует потоки вместо процессов
Создание процесса — дорогая операция. Для матрицы 50×50 это 2500 процессов. Pool создаёт, например, 4 процесса и распределяет все 2500 задач между ними, что значительно эффективнее.
Вопрос 7

Что определяет конструкция async def в Python?

A Отдельный поток
B Новый процесс ОС
C Корутину (асинхронную функцию)
D Генератор
async def определяет корутину — специальную функцию, которая может приостанавливать своё выполнение в точках await и возобновляться позже.
Вопрос 8

Что происходит при выполнении await внутри корутины?

A Программа завершается
B Создаётся новый поток для выполнения операции
C Текущая корутина приостанавливается, управление передаётся event loop
D Запускается новый процесс
await приостанавливает текущую корутину и передаёт управление event loop. Пока корутина ожидает (например, сетевого ответа), event loop может выполнять другие корутины.
Вопрос 9

Что произойдёт, если в асинхронной функции использовать time.sleep(5) вместо await asyncio.sleep(5)?

A Ничего — результат одинаковый
B Программа аварийно завершится с ошибкой
C Event loop заблокируется на 5 секунд, все остальные корутины встанут
D Будет создан отдельный поток для ожидания
time.sleep() — блокирующая операция. Она останавливает весь поток, включая event loop. Все остальные корутины не смогут выполняться эти 5 секунд. asyncio.sleep() отпускает управление event loop.
Вопрос 10

Что делает asyncio.gather()?

A Запускает корутины последовательно
B Запускает несколько корутин конкурентно и ожидает завершения всех
C Собирает результаты из нескольких потоков
D Создаёт новый event loop для каждой корутины
asyncio.gather() принимает несколько корутин и запускает их конкурентно в одном event loop. Возвращает список результатов, когда все корутины завершены.
Вопрос 11

Сколько потоков использует asyncio по умолчанию?

A Столько, сколько ядер CPU
B Два (основной + event loop)
C Один
D asyncio использует процессы, а не потоки
asyncio работает в одном потоке. Конкурентность достигается за счёт кооперативного переключения между корутинами в точках await, а не за счёт параллельного выполнения.
Вопрос 12

Какой тип многозадачности реализует asyncio?

A Вытесняющая (preemptive)
B Параллельная (parallel)
C Кооперативная (cooperative)
D Конкурентно-параллельная
В asyncio переключение между задачами происходит только в точках await — программист сам определяет, где корутина может быть приостановлена. Это кооперативная многозадачность.
Вопрос 13

Что вернёт os.getpid() в дочерних процессах multiprocessing?

A Тот же PID, что и у родителя
B Всегда 0
C Уникальный PID для каждого дочернего процесса
D PID родительского процесса
Каждый процесс ОС получает уникальный PID. В multiprocessing дочерние процессы — это настоящие процессы ОС с собственными PID, в отличие от потоков, которые разделяют PID своего процесса.
Вопрос 14

Зачем в скриптах с multiprocessing нужна конструкция if __name__ == '__main__'?

A Это просто стилистическое соглашение PEP 8
B Она ускоряет выполнение
C Она предотвращает повторное выполнение кода верхнего уровня в дочерних процессах
D Её требует операционная система
При создании дочернего процесса Python импортирует модуль заново. Без этой проверки код создания процессов выполнится и в дочернем процессе, вызвав бесконечную рекурсию.
Вопрос 15

Три IO-операции занимают 2, 3 и 1 секунду. Сколько времени займёт их выполнение через asyncio.gather()?

A 6 секунд (2 + 3 + 1)
B 2 секунды (время самой быстрой)
C ≈3 секунды (время самой долгой)
D Зависит от количества ядер CPU
asyncio.gather() запускает все корутины конкурентно. Пока одна ожидает IO, другие выполняются. Общее время определяется самой долгой операцией — max(2, 3, 1) = 3 секунды.
Вопрос 16

Что делает await writer.drain() в asyncio?

A Удаляет объект writer
B Читает данные от клиента
C Ожидает, пока буфер записи будет отправлен по сети
D Закрывает сетевое соединение
writer.write() помещает данные в буфер, но не гарантирует немедленную отправку. drain() — это корутина, которая ожидает, пока данные из буфера действительно уйдут в сеть.
Вопрос 17

Чем отличается вывод PID в многопроцессном сервере от многопоточного?

A Разницы нет
B В threading у каждого обработчика свой PID
C В multiprocessing каждый обработчик имеет уникальный PID, а в threading — все имеют одинаковый PID
D В multiprocessing PID не отображается
Потоки (threading) выполняются внутри одного процесса ОС, поэтому os.getpid() возвращает один и тот же PID. Процессы (multiprocessing) — это отдельные процессы ОС, каждый с уникальным PID.
Вопрос 18

Что делает Pool.starmap(func, iterable)?

A Вызывает func один раз с одним аргументом
B Создаёт новый процесс для каждого вызова
C Распределяет вызовы func с несколькими аргументами по процессам пула
D Останавливает все процессы пула
starmap() — аналог map(), но позволяет передавать несколько аргументов каждому вызову. Он распределяет задачи между существующими процессами пула, не создавая новых.
Вопрос 19

Для веб-сервера, обслуживающего тысячи одновременных подключений, какой подход наиболее эффективен?

A multiprocessing — по процессу на клиента
B threading — по потоку на клиента
C asyncio — конкурентная обработка в одном потоке
D Последовательная обработка
Обработка сетевых подключений — IO-bound задача. asyncio-корутины легковесны (можно создать тысячи), не требуют переключения контекста ОС и минимально потребляют ресурсы.
Вопрос 20

Какой вызов корректно создаёт и запускает TCP-сервер в asyncio (Python 3.8+)?

# Вариант A: loop = asyncio.get_event_loop() loop.run_until_complete(asyncio.start_server(handler, host, port, loop=loop)) # Вариант B: server = await asyncio.start_server(handler, host, port) async with server: await server.serve_forever()
A Вариант A — он более явный и надёжный
B Оба варианта равнозначны
C Вариант B — это современный стиль Python 3.8+; параметр loop= удалён в Python 3.10
D Ни один — нужно использовать socket.socket()
Вариант A использует устаревший API (параметр loop= deprecated с Python 3.8 и удалён в 3.10). Вариант B использует asyncio.run() + async with — это современный, рекомендованный стиль.
